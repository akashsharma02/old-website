NeRF: Representing Scenes as Neural Radiance Fields for View Synthesis

Abstract

  • Task: Novel View Synthesis by optimizing volumetric radiance function using "sparse" input views
  • Scene representation: Fully connected neural network with inputs: (x,y,z)(x, y, z)(x,y,z)ï»¿ spatial position and (Î¸,Ï•)(\theta, \phi)(Î¸,Ï•)ï»¿ viewing direction, and outputs: volume density and view-dependent emitted radiance.
    • Note: spatial position and viewing direction defines one ray in the volume, and the network outputs volume density distribution over the ray, and a radiance
  • Renders are photorealistic

Introduction

  • Static scene is represented as a continuous 5D function:
  • To render this function aka Neural Radiance Field (NeRF) from a particular viewpoint:
    1. Given a novel camera view, march ray for each pixel of the image, and generate a sampled set of 3D points (sampling is done over the entire camera ray). We have many 3D points, each with the same viewing direction per ray.
    1. Use these sampled points and their corresponding viewing direction as input to the network to obtain their colors and densities.
    1. Finally compose the image, by projecting along the ray the accumulated color and radiance into each pixel of the novel view
  • Note: This requires a bounded volume for ray to terminate
  • input 5D coordinates are positional encoded using sinusoidal encoding for better sample efficiency and learning higher resolution representation. Also a small bit of cheating via stratified sampling is required.
